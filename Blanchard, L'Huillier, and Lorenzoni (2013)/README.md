# Blanchard, L'Huillier, and Lorenzoni (2013)

This directory is a Dynare 7.1 port of the replication files for [“News, Noise, and Fluctuations: An Empirical Exploration”](https://www.aeaweb.org/articles?id=10.1257/aer.103.7.3045). It replaces the matrix expressions accepted by old Dynare releases with a scalar model plus MATLAB helper functions. It does not patch Dynare's `stoch_simul` or estimation routines.

## Main repair

The external steady-state file now starts from `M_.params` and changes only the 16 coefficients of the signal-extraction matrices `BM`, `FA`, and `CM`. These coefficients are recomputed from the current candidate values of `rho`, `sig_u`, and `sig_nu` at every likelihood evaluation. No calibrated assignment in the steady-state file can overwrite an estimated parameter.

The estimation and simulation files also include the same model source, `Model/bll_model.inc`. In particular, the wage Phillips-curve slope is evaluated directly from the current `cal_w` and `zet` values. The old port fixed `kap_w` before estimation, so those candidate parameters did not update the slope.

## Two specifications

The authors' computational files and the published paper/online appendix are not identical. The default profile follows the computational implementation because that is the profile that reproduces the reported posterior units and results:

- the resource constraint uses the authors' `1.28` consumption-output calibration;
- price and wage markup shocks enter the Phillips curves in reduced-form units (`+m_p` and `+m_w`);
- priors follow the computational files where they conflict with Table 5.

Set `PAPER_SPECIFICATION=1` to use the online appendix resource constraint, the appendix markup-shock scaling, and the priors printed in Table 5. This is an audit profile, not a route to numerically reproduce estimates generated by the computational profile.

The paper's Table 6 has a further labeling problem. The nontechnology columns follow the old positional shock order, while their printed headings say Price, Wage, and Monetary. Outputs from this port use the declared shock names and therefore report the semantic order: investment-specific, price markup, wage markup, monetary, fiscal.

## Requirements

- MATLAB R2026a (the code also uses only broadly available recent MATLAB features)
- Dynare 7.1

Add Dynare to the MATLAB path before running the commands below. On Apple Silicon with the standard Dynare installer:

```matlab
addpath('/Applications/Dynare/7.1-arm64/matlab')
```

## Estimation workflow

Run these commands from `Estimation`.

1. Find the posterior mode:

   ```matlab
   dynare estimate.mod -DMH_REPLIC=0 -DMODE_COMPUTE=6
   ```

2. Prepare the Metropolis proposal:

   ```matlab
   prepare_mode_file
   ```

   The mode for `gam_pi` is close to the determinacy boundary. Dynare's numerical Hessian consequently has one non-positive eigenvalue in the tested environment. This script regularizes only the proposal Hessian; it does not change the mode, likelihood, priors, or target posterior.

3. Run a smoke chain or the full replication chain:

   ```matlab
   dynare estimate.mod -DMODE_COMPUTE=0 -DUSE_MODE_FILE=1 -DMH_REPLIC=4000
   dynare estimate.mod -DMODE_COMPUTE=0 -DUSE_MODE_FILE=1 -DMH_REPLIC=120000
   ```

   The Dynare 7.1 proposal scale defaults to `0.01`. Override it with, for example, `-DMH_JSCALE=0.008`. Add `-DCOMPUTE_POSTERIOR_IRFS=0` when tuning chains and no IRF draws are needed.

4. Transform and plot posterior IRFs:

   ```matlab
   make_IRFs
   ```

`make_IRFs.m` loads paired parameter and reduced-form IRF draws using Dynare's public posterior-IRF interface. It performs the permanent/transitory/noise transformation separately for every draw, then computes posterior means, medians, 90% empirical HPD intervals, and Table 6 variance shares. Transforming only an already averaged IRF is not equivalent and is intentionally avoided.

Use `-DPAPER_SPECIFICATION=1` on the Dynare command to run the audit profile. `-DRUN_ESTIMATION=0` performs preprocessing and model diagnostics without optimizing or sampling.

## Posterior-mean simulation

Run this from `Simulation`:

```matlab
IRF_convert
```

`simulate.mod` uses the posterior means reported in Table 5. `IRF_convert.m` transforms the two reduced innovations into the three structural technology shocks and exports the Figure 5 and Figure 6 counterparts.

## Tests

From MATLAB:

```matlab
run('../tests/run_tests.m')
```

The fast suite checks the Kalman/Riccati solution against the legacy algorithm, covariance-factor validity, candidate-parameter preservation, invalid-candidate rejection, structural IRF dimensions and bands, the dynamic wage slope, shared model inclusion, and both specification profiles. The port has also been exercised through Dynare 7.1 preprocessing, steady-state solution, determinacy checks, mode finding, two-block Metropolis sampling, and draw-by-draw posterior IRF construction.

Dynare may warn that `sig_u` and `sig_nu` are unused. That is a preprocessor limitation: both enter through the external steady-state function. In the default computational profile, `psi` is also intentionally unused because the authors' resource equation uses `cy_ratio` instead.

Generated Dynare folders, posterior draws, figures, tables, logs, and intermediate MAT files are ignored by Git. `Estimation/data.mat` is source data and remains versioned.

## Source comparison

The paper, its [online appendix and replication materials](https://www.aeaweb.org/articles/materials/2517), and the original computational files were compared equation by equation. The specification switch exists so the discrepancies remain visible rather than being silently “corrected” in one direction.
